# Phase 4: Engagement & Assessment Design

## Purpose
Design the systems that keep students engaged, accountable, and completing the course.

---

## Onboarding Experience

### Welcome Email Sequence (3 emails)

**Email 1 — Immediately after purchase: Welcome + What to Expect**
- Confirm enrollment
- Set expectations (time commitment, format, community)
- Provide access credentials/link
- Quick win: one simple task to do RIGHT NOW

**Email 2 — Day before start: Getting Ready**
- Tech setup checklist
- Join the community
- Meet your accountability partner
- Preview Day 1

**Email 3 — Day of start: Let's Go!**
- Link to Day 1 lesson
- Reminder of the transformation
- Community call-to-action

### Orientation Lesson (Lesson 0)
- Welcome video from instructor (2-3 min, personal, energetic)
- Course roadmap: "Here's exactly what we'll cover and when"
- How to use the platform
- How the community works
- Introduce the accountability system
- Set personal goals (worksheet)
- Post an intro in the community

---

## Accountability Mechanics

| Mechanism | How It Works | Best For |
|---|---|---|
| Daily check-in | Post "Done!" or share work daily | Challenges |
| Accountability partner | Paired up, check in with each other | Cohort + Challenge |
| Progress tracker | Visual tracker (checklist/progress bar) | All formats |
| Streak counter | Track consecutive days of completion | Challenges |
| Weekly standup | Share: what I did, what I learned, what I'm stuck on | Cohort |
| Public commitment | Post goals publicly at start | All formats |

Let user choose which mechanisms fit their course format and audience.

## Milestone Celebrations

| Milestone | Celebration |
|---|---|
| Complete onboarding | Welcome badge + community shoutout |
| First module complete | Certificate of progress + instructor congrats |
| Halfway point | Special live session or bonus content |
| All assessments passed | Achievement badge |
| Course complete | Certificate of completion + showcase opportunity |
| Capstone submitted | Feature in community/social media |

## Assessment Design

### Quiz Template
```markdown
# QUIZ: [Module X] Knowledge Check

- 5-7 multiple choice or true/false questions
- Aligned to learning objectives (Remember/Understand level)
- Immediate feedback on each answer
- Pass threshold: 70%
- Retake allowed: Yes (unlimited)

## Questions

### Q1: [Question text]
- a) [Option]
- b) [Option]
- c) [Option] ✓
- d) [Option]

**Feedback:** [Explanation of correct answer]
```

```markdown
- [Objective 2]

## Requirements
- [ ] [Specific requirement 1]
- [ ] [Specific requirement 2]
- [ ] [Specific requirement 3]

[How to submit — link, file upload, community post]

## Evaluation Criteria
- [Criterion 1]: [Description]
- [Criterion 2]: [Description]
- [Criterion 3]: [Description]

## Due Date
[If cohort/challenge]

## Peer Review (if applicable)
Review 2 other students' submissions using the provided rubric.
```

### Capstone Project Template
```markdown
# CAPSTONE PROJECT: [Course Name]

## Overview
This is your chance to demonstrate the full transformation.
[Description of what they build]

## Requirements
[Comprehensive list]

## Timeline
- Week X: Project proposal
- Week Y: Check-in / draft review
- Week Z: Final submission

## Showcase
Present your capstone in the final live session / community showcase event.
```

## Live Element Planning (Cohort/Challenge)

| Session | Frequency | Duration | Format |
|---|---|---|---|
| Kickoff call | Once (Day 1) | 60 min | Welcome + orientation + Q&A |
| Weekly Q&A | Weekly | 45-60 min | Open questions + hot seats |
| Office hours | Bi-weekly | 30-45 min | Drop-in help |
| Mid-course check-in | Once (midpoint) | 30 min | Progress review + motivation |
| Showcase/graduation | Once (final) | 60-90 min | Student presentations + celebration |
