# Reference

External knowledge, tools documentation, approaches, and resources.

## What Goes Here
- Notes from books, courses, podcasts
- Tool documentation and how-to guides
- Methodologies and frameworks you follow
- Links and bookmarks organized by topic
- The setup guide for this vault

## Suggested Subfolders
- `tools/` — Documentation for tools you use
- `books/` — Book notes and summaries
- `courses/` — Course notes
- `articles/` — Saved articles and highlights
- `guide/` — Setup guide for this vault

## Naming Convention
`Source - Title.md` or just `Title.md`

## Frontmatter
```yaml
---
type: reference
source: book | course | article | tool | other
title: Resource Title
author: Author Name
date: YYYY-MM-DD
tags: [topic1, topic2]
---
```

## Tips
- Reference notes are external knowledge; thinking notes (Thinking) are your synthesis
- After reading a reference, write a thinking note about what you took from it
- Link reference notes to projects they're relevant to
